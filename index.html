1. What is a Database ?
  • It is a collection of related data stored in an organised way to be easily accessed and manipulated. For Example - set of student information.
    • 2. What is a Database Management System ?
  • It is a software used to manage Database in a more efficient way. For Example - MySQL, Oracle etc. In simple terms, it is the interface between the users and the Database files.
    • 3. What is a Relational Database Management System ?
  • It is a type of database management system whose data is stored in the form of tables. It includes functions to that maintains the integrity, consistency and accuracy of data. For Example - Oracle, MySQL, IBM DB2 etc

    • 6. What is DDL, DML, DCL ?
  • A DDL is used to define/modify the structure/definition of a database table and also used to modify the Integrity constraints. For example: CREATE, DROP, ALTER etc.
  • A DML is used to modify/access the data records from a database table. For example: INSERT, UPDATE, DELETE etc.
  • A DCL is used for data control transaction management and security/access control. For example: COMMIT, ROLLBACK etc.

    • 7. What is the difference between DBMS and RDBMS ?
  • DBMS allows one user at a time, while RDBMS allows multiple users at a time.
  • DBMS stores data in hierarchal form whereas an RDBMS stores data in the form of tables.
  • DBMS does not support integrity constraints while RDBMS supports the integrity constraints.
  • DBMS does not support Normalization, while RDBMS does.
  • In DBMS fetching data is slow, while it is fast in RDBMS because of its relational approach.
  • DBMS does not obey ACID properties while RDBMS do follow.

    • 10. What are ACID properties ?
  • Atomicity: Any transaction in a database might have multiple sub transaction statements. The Atomicity property ensures if any of the sub transaction statements fails then the entire transation should fail and the database state is left unchanged, i.e., all transactions and the sub transaction statements should either succeed completely or fails but not partially.
  • Consistency: It ensures that the data must meet all the data validations rules, constraints and every transaction should result in leaving the database in a stable and consistent state.
  • Isolation: It ensures concurrency control i.e. the concurrent execution of transactions should leave the database in the same state that would have been obtained if the transactions were executed sequentially
  • Durability: It ensures after a successful transaction is completed, all the changes perists regardless of any system failure.

    • 11. What is view in SQL. How is it different from a table ?
  • Views are virtual tables (definitions) that are built on top of the actual table. Views can be created on top of oneor more tables or views.

    • CREATE VIEW EMP AS
    SELECT EMPID,EMPNAME,DEPTID FROM EMPLOYEE
    ORDER BY EMPID

    Once the view is created, we can query it like a normal table as below -

    SELECT * FROM EMP

  • Views do not hold the records themselves, any changes in the underlying table records will be reflected in the view
  • Views are faster than the actual table in terms of performance, since it does not contain the underlying data and fetching records is much easier. It does not use any physical memory.

    • 12. What are NESTED Queries ?
  • A query within another query is known as nested query. It is of two types:

    • a. NESTED QUERY WITH CORELATION - A nested query where the inner query uses attributes defined in outer query. It is executed once for each row processed by outer query.
    SELECT ID,NAME,SALARY FROM EMPLOYEE AS E WHERE SALARY >
    (SELECT AVG(SALARY) FROM EMPLOYEE WHERE department=E.department)

    (The above query outputs the name,id and salary of all employee whose salary is greater than the average salary of all the employees working in the same department)

    b. NESTED QUERY WITHOUT CORELATION - A nested query where the inner query runs independently. The inner query executes first and it passses the results to the outer query.

    13. What are Aggregate Functions

  • Aggregate functions in SQL are used to group together multiple rows and returns a single value.
  • Aggregate functions computes aggregation of non NULL values. NULL values are not considered for aggregation.
  • Some aggregate functions are MIN,MAX,COUNT,AVG etc
  • The WHERE clause cannot be used on Aggregation (but can be used to filter rows before grouping) because the WHERE clause is executed first and it will result in an error. To avoid this, we use HAVING clause to filter aggregated data.
  • The GROUP BY clause treats NULL as a single value and aggregates for the set.

    • 14. What is GROUP BY Clause
  • A GROUP By clause is used to group together records based on specific attributes.
  • Every attribute of a GROUP BY Clause must appear in the SELECT Clause.

    • SELECT A,B
    FROM EMPLOYEE
    GROUP BY A

    CORRECT !
    SELECT B
    FROM EMPLOYEE
    GROUP BY A

    INCORRECT !

  • Aggregated function used in SELECT Clause is computed for each group.

    • 15. What is HAVING Clause

  • A HAVING clause is used to select groups based on specific condition.
  • HAVING clause must be followed by a GROUP BY clause.
  • A HAVING clause must be over aggregation or over SOME(), EVERY(), ALL().

    • SELECT SALARY
    FROM EMPLOYEE
    GROUP BY SALARY
    HAVING SOME(SALARY) > 10000

  • In a HAVING clause direct attribute comparison will not work.

    • SELECT SALARY
    FROM EMPLOYEE
    GROUP BY SALARY
    HAVING SALARY > 10000

    INCORRECT !

    16. What is ORDER BY Clause ?
  • A ORDER BY clause is used to sort the data in ASCENDING order (by default or use ASC keyword) or DESCENDING order (use DESC keyword)
  • In an ORDER BY clause you can sort data based on one or more column.
  • In the below example, the result is first sorted on the basis of NAME coulmn in ASCENDING order and then by AGE in DESCENDING order
    • SELECT *
    FROM EMPLOYEE
    ORDER BY NAME, AGE DESC
