1. What are Constraints ?

Constraints are like rules used to restrict the type of data inserted into the table, this ensures accuracy and reliability of data in the database.

  • NOT NULL: To restrict NULL values.
  • PRIMARY KEY: To uniquely identify each row in a table.

    • CREATE TABLE EMPLOYEE (
    EMPID int(10) PRIMARY KEY,
    NAME varchar(50),
    PHONE int(10) NOT NULL )

  • FOREIGN EKY: To uniquely identify each row in another table

    • CREATE TABLE EMPLOYEE (
    EMPID int(10) PRIMARY KEY,
    NAME varchar(50),
    PHONE int(10) NOT NULL )


    CREATE TABLE COMPANY (
    CID int(10) PRIMARY KEY,
    CNAME varchar(50),
    FOREIGN KEY ID REFERENCES EMPLOYEE(EMPID) )

  • CHECK: Ensures that the values of the column meet a particular condition.

    • CREATE TABLE EMPLOYEE (
    EMPID int(10) PRIMARY KEY,
    NAME varchar(50),
    AGE int CHECK (AGE>18))

  • UNIQUE: To allow only unique values in a column.
  • DEFAULT: It specifies a default value for that column.

    • CREATE TABLE EMPLOYEE (
    EMPID int(10) PRIMARY KEY,
    NAME varchar(50),
    AGE int DEFAULT 18 )
