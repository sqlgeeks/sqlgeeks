1. What are JOINS in SQL?
  • JOINS are used to combine two or more tables by making use of some related column
  • INNER JOIN is used to retrieve the matching data between the two tables

    • SELECT STUDENT.Name, EMPLOYEE.Salary FROM STUDENT INNER JOIN EMPLOYEE ON STUDENT.Id=EMPLOYEE.Id
    (The above query returns records for the Id's in STUDENT table that matches with the Id's in the EMPLOYEE table)

  • LEFT JOIN is used to retrieve all the data from the left table and the matching data from the right tables

    • SELECT STUDENT.Name, EMPLOYEE.Salary FROM STUDENT LEFT JOIN EMPLOYEE ON STUDENT.Id=EMPLOYEE.Id

  • RIGHT JOIN is used to retrieve all the data from the right table and the matching data from the left tables.

    • SELECT STUDENT.Name, EMPLOYEE.Salary FROM STUDENT RIGHT JOIN EMPLOYEE ON STUDENT.Id=EMPLOYEE.Id

  • FULL JOIN or FULL OUTER JOIN is used to retrieve all the data from both the tables including the unmatching rows. The unmatched columns are filled with NULLS.
  • FULL JOIN is majorly used to understand the overlap between the two tables.

    • SELECT STUDENT.Name, EMPLOYEE.Salary FROM STUDENT FULL OUTER JOIN EMPLOYEE ON STUDENT.Id=EMPLOYEE.Id


  • SELF JOIN is just like a simple join, where a table is joined with itself. Self Joins are useful for comparisons within table.

    • 2. What is the difference between Cross Join and Natural Join ?
  • A CROSS JOIN outputs the cartesian product of two tables i.e. The number of rows in the first table multiplied by the number of rows in the second table produces the Cartesian Product and hence are CROSS JOIN.
  • An interesting part about using CROSS JOIN is that when a WHERE clause is used with CROSS JOIN, it acts as a INNER JOIN
  • A NATURAL JOIN on the other hand joins two tables based upon same column names and datatypes.
