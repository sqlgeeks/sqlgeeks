1. What is INDEX in SQL ?

- INDEXES are used to speed up the retrieval of rows from the table or view.
- INDEXES are helpful in case where there are thousands of records and we need to find records for one or more combination of columns. The normal way takes a longer time but with the help of indexes the records can be fetched very quickly.

CREATE INDEX EMP ON EMPLOYEE(EMPLOYEE_CITY)

CREATE INDEX EMP ON EMPLOYEE(EMPLOYEE_CITY,EMPLOYEE_COUNTRY)
(Creating Index using one or more columns).

- There are two types of INDEXES -

a. Clustered Index: A clustered index physically orders the data according to the columns of an index. There can only be one clustered index on a table (applying a PRIMARY KEY to a column makes it a Clustered Index ) as the rows in the table will be sorted by the order specified in the clustered index. The Clustered indexes are stored as trees where the actual data is stored in the leaf nodes, speeding up the lookup process.

b. Non-Clustered Index: A non-clustered index also know as B-Tree index are created on top of the actual data. Unlike clustered index, the leaf does not contain the actual data rather the leaf pages of the non clustered index contain the pointers pointing to the clustered index leaf nodes making it slower.(It required some additional disk space)

You use Clustered Index when you are always required to return data in an ordered manner. Here you can create a clustered index with the required columns of the ORDER BY clause and all the rows of the table are not required to be scanned.

You use Non-Clustered Index when you want to frequently return records in a specific manner. When you are using a certain column more frequently in the queries creating a Non-Clustered Index will speed up the performance and retrieval.
